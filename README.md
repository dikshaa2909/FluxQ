# FluxQ ― Intelligent Queue Optimization Ecosystem

◆ **Project Overview**  
FluxQ is a full-stack solution for clinical triage and telemedicine queue management. It combines a high-performance logic engine with a real-time monitoring dashboard to reduce patient wait times and optimize healthcare delivery.

◆ **Project Structure**

◈ **[FluxQ Dashboard (Client)](file:///client/README.md)**  
A React 19 / TypeScript dashboard featuring specialized clinical intake, live simulation, and deep performance analytics.  
▶ **Live URL**: [flux-q.vercel.app](https://flux-q.vercel.app)

◈ **[FluxQ Engine (Server)](file:///server/README.md)**  
The core REST API and logic engine. Available as a standalone NPM package for integration into any healthcare platform.  
▶ **API Endpoint**: [fluxq.onrender.com](https://fluxq.onrender.com)  
▶ **NPM Package**: [fluxq-engine](https://www.npmjs.com/package/fluxq-engine)

◆ **Core Philosophy**
• **Clinical Accuracy**: Prioritizes patients based on acute-specific severity multipliers.
• **Operational Fairness**: Prevents starvation through dynamic aging boosts.
• **Visual Intelligence**: Real-time feedback via high-end glassmorphism components.
